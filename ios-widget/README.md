# 아이폰 홈 화면 위젯 (Scriptable)

이 웹앱은 PWA라서 "앱처럼" 설치는 되지만, iOS 홈 화면 위젯(WidgetKit)은 네이티브 앱만 만들 수 있어서
웹사이트만으로는 직접 지원할 수 없어요. 대신 무료 앱 **Scriptable**을 이용해서, 같은 Firebase 계정의
데이터를 읽어와 위젯을 만들 수 있어요. 현재 두 가지가 있어요.

| 스크립트 | 내용 | 추천 크기 |
| --- | --- | --- |
| `today-schedule-widget.js` | 오늘의 일정 목록 | Small / Medium |
| `month-calendar-widget.js` | 이번 달 전체 캘린더 (일정 있는 날 점 표시) | Large (필수) |

두 스크립트는 완전히 독립적이라 하나만 설치해도 되고, 둘 다 설치해서 다른 위젯으로 각각 추가해도 돼요.
로그인 정보(이메일/비밀번호)는 아래 3번 과정을 스크립트마다 한 번씩 해주면 돼요 (같은 키체인 키를 쓰지만,
Scriptable 스크립트는 서로 독립 실행되기 때문에 각자 최초 1회 로그인이 필요해요).

## 1. Scriptable 설치

App Store에서 **Scriptable** (Simon B. Støvring 제작, 무료)을 설치하세요.

## 2. 스크립트 추가

1. Scriptable 앱을 열고 오른쪽 위 **+** 버튼으로 새 스크립트를 만듭니다.
2. 원하는 스크립트 파일(`today-schedule-widget.js` 또는 `month-calendar-widget.js`) 내용을 전부
   복사해서 붙여넣습니다.
3. 왼쪽 위 이름을 눌러 알아보기 쉬운 이름으로 바꿔줍니다. 예: **"오늘의 일정"**, **"이번 달 캘린더"**.

## 3. 로그인 정보 저장 (스크립트마다 한 번씩)

1. 스크립트 편집 화면에서 하단 재생 버튼(▶)을 눌러 **앱 안에서 직접 한 번 실행**합니다.
2. 대시보드 로그인 이메일/비밀번호 입력창이 뜨면 입력하고 저장을 누르세요.
   - 이 정보는 이 아이폰의 **키체인(Keychain)**에만 저장돼요. 스크립트 코드 안에는 저장되지 않아요.
   - 위젯은 화면에 직접 입력창을 띄울 수 없어서, 이렇게 앱에서 미리 한 번 저장해둬야 위젯이 자동으로 로그인할 수 있어요.
3. 이후 정상적으로 미리보기가 뜨면 성공입니다.

비밀번호를 바꿨거나 로그인 정보를 다시 입력하고 싶으면, 앱에서(위젯이 아니라) 스크립트를 다시 실행하면
"로그인 정보 재설정" 확인창이 떠요. 거기서 "다시 입력"을 누르면 다음 실행 때 새로 입력할 수 있어요.

## 4. 홈 화면에 위젯 추가

1. 홈 화면 빈 곳을 길게 눌러 **+** 버튼(오른쪽 위)을 누릅니다.
2. **Scriptable** 검색 → 원하는 크기 선택 → 위젯 추가.
   - `today-schedule-widget.js`는 Small 또는 Medium 추천.
   - `month-calendar-widget.js`는 **Large**를 선택해야 캘린더 그리드가 제대로 보여요. Small/Medium으로
     추가하면 "Large로 추가해주세요" 안내만 표시돼요.
3. 추가된 위젯을 길게 눌러 **위젯 편집**을 선택합니다.
4. **Script**: 방금 만든 스크립트 선택
5. **When Interacting**: `Run Script` (위젯을 탭하면 대시보드 사이트가 열려요)
6. **Parameter**: 비워둡니다.

이제 위젯이 15~30분 간격(iOS가 자동으로 정하는 주기)으로 새로고침돼요.
- **오늘의 일정**: 카테고리 색 점 + 일정 제목, 완료 처리한 일정은 흐리게 표시.
- **이번 달 캘린더**: 요일 헤더 + 6주 그리드, 오늘 날짜는 강조 배경, 일정이 있는 날짜엔 점 표시,
  하단에 이번 달 총 일정 개수 표시.

## 참고

- 두 스크립트 모두 대시보드와 같은 Firebase 프로젝트(personal-assistant-ec8b5)의 Firestore를 REST
  API로 직접 읽기만 합니다. 쓰기(수정/삭제)는 하지 않아요.
- 로그인 이메일/비밀번호는 이 아이폰의 Keychain에만 저장되고, 매 새로고침마다 Firebase에 재로그인해서
  토큰을 새로 받아요 (토큰을 따로 저장/갱신하지 않아 구조가 단순해요).
- `month-calendar-widget.js`의 셀 크기(`CELL_W`/`CELL_H`)는 기기 화면에 맞춰 대략적으로 잡은 값이라,
  기기에 따라 살짝 넘치거나 여백이 남을 수 있어요. 그럴 땐 스크립트 상단의 `CELL_W`/`CELL_H`/`COL_GAP`/
  `ROW_GAP` 값을 조금씩 줄이거나 늘려서 맞추면 돼요.
- "연습 현황"이나 "운동 기록" 같은 다른 위젯도 같은 방식으로 만들 수 있어요. 필요하면 요청하세요.
